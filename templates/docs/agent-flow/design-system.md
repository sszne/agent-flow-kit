# Design System

Use this optional document to record durable design-system guidance for
frontend planning. `flow-plan` reads this document when screens, components,
frontend routes, client UI, styles, public frontend assets, brand, tokens, or
component rules are in scope.

## Source Priority

```text
explicit user-provided design-system docs > repo-local design-system docs >
existing source/components/styles > general design taste
```

When this document conflicts with current source components or styles, record
the conflict and ask for confirmation before treating either source as
authoritative.

## Waiver Rules

Waivers require a concrete reason, such as legacy component incompatibility,
accessibility improvement, missing token guidance, conflict with current source
evidence, explicit user instruction, or no design-system artifact found after
searched-path review.

Invalid waivers: `N/A`, `manual`, `low risk`, `TBD`, `later`, blank, or a
generic "not applicable" without searched-path evidence.
